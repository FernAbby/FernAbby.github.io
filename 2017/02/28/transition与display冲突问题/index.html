transition与display冲突问题

发表于 2017-02-28 | 分类于 CSS | | 阅读次数

元素使用’display:none（block）’实现隐藏（显示）时，CSS3的transition动画效果无效。至于产生这种现象的深层次的机制原因不清楚，暂时理解为CSS3的transition过渡不支持display的改变，直接操作display会破坏transition的动画。

解决方案:

1. visiblity代替display

2. setTimeout

弊端：不同浏览器delay不同，如，chrome与IE10+可以为0，但是FF必须delay>=14

3. window.requestAnimationFrame

原理与setTimeout相同

4. 中间穿插代码，跳出当前处理tick机制（也不知怎么解释这个，暂时这么说吧）

个人理解：display的操作会触发浏览器的reflow操作，而transition支持的效果只是触发浏览器的repaint操作，回到上面的demo，如果我们通过visibility属性来控制显示与隐藏，则不会破坏transition的效果。所以，可以暂时这么认为：reflow与repaint的混合会破坏transition的动画效果，
